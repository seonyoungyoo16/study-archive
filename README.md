# 🌱 Junior Backend Developer Study Archive

> **꾸준히 성장하는 백엔드 개발자가 되기 위해 학습한 이론과 실습 코드를 기록하는 공간입니다.**
> 
> 단순 코드 복사 대신 동작 원리 이해와 문제 해결 과정을 중심으로 기록하며, 배운 내용을 실무와 토이 프로젝트에 적용하는 것을 목표로 합니다.

---

## 📅 공부 규칙 (Commit Strategy)
* **매주 최소 3회 이상** 학습한 내용을 정리하거나 실습 코드를 커밋합니다.
* 커밋 메시지는 `feat:`, `docs:`, `fix:` 등 컨벤션을 준수하여 가독성을 높입니다.

---

## 📂 카테고리별 학습 내용

### 🟢 Java & Spring Framework (`/java-spring`)
* **목적:** 스프링 핵심 원리를 이해하고 효율적인 비즈니스 로직을 구현합니다.
* **주요 내용:** JPA N+1 문제 해결 및 쿼리 최적화, JWT 기반 인증/인가 프로세스 구현.

### 🔵 Database (`/database`)
* **목적:** 데이터 정합성을 지키고 대용량 데이터를 효율적으로 다루는 방법을 배웁니다.
* **주요 내용:** SQL 인덱스 튜닝을 통한 조회 성능 개선 실험, 트랜잭션 격리 수준별 부작용(Dirty Read 등) 재현 및 해결.

### 🔴 Data Structure & Algorithm (`/data-structure-algorithm`)
* **목적:** 복잡한 문제를 논리적으로 해결하는 컴퓨팅 사고력을 기릅니다.
* **주요 내용:** 백준 및 프로그래머스 레벨 2~3 단계 문제 풀이 (주 2회 업로드).

### 🟡 Computer Science (`/CS-networks`)
* **목적:** 면접 대비 및 네트워크/OS 기술 면접 질문에 깊이 있게 답변하기 위한 기반 지식을 다집니다.
* **주요 내용:** HTTP/HTTPS 프로토콜의 차이, 프로세스와 스레드의 자원 공유 방식 정리.

## 📝 실습 기록 양식
모든 실습 코드가 포함된 폴더에는 아래 내용을 간략히 기록합니다.
1. **무엇을 유도했는가?** (예: JPA Fetch Join을 통한 쿼리 수 감소)
2. **발생한 에러/이슈와 해결 과정** (삽질 기록)
3. **배운 점**
